Plan: Steelman Agent Visual Vocabulary Expansion

Feature · Medium-Complex

Task Description

The goal is to expand the Steelman agent's visual vocabulary with three new high-impact artifact types:

  1. Quote Artifact (quote): A typography-forward callout to render critical statements (e.g., from earnings calls, reports, or regulators). It must feature a large opening quotation mark, italicized body, and a clean attribution line with an inline link, with space for a short-thesis commentary block.
  2. Catalyst Timeline Artifact (catalyst-timeline): A chronological left-to-right chain of dated events (e.g., covenant breach, ratings downgrade) color-coded by severity (bull/neutral/bear) to visualize pacing and sequence.
  3. Valuation Gauge Artifact (valuation-gauge): A horizontal scale that places price targets (bear, base, bull) in relation to the current stock price, providing a quick read on market positioning.

Objective

Equip the Steelman agent with richer visualization options that make structural bear theses and market mispricings instantly legible. The changes span the web frontend (rendering), product-specific TypeScript agent extensions (parameter typing/schema validation), and LLM system prompt instructions (instructing the agent on how/when to use them).

Problem Statement

Prose and tabular data are excellent for presenting arguments, but key risk mechanisms in complex investment structures often get lost in translation. Short sellers and hedge funds rely heavily on three analytical structures that currently require manual generation or are flat-out impossible to render using existing generic components (like simple bar/pie charts):

  • Smoking gun admissions: Hard quotes from management are the highest form of evidence; they deserve premium typography rather than getting buried in blocks of text.
  • Sequence of failure: Catalysts represent dynamic domino chains where timing and severity dictate structural breakdowns. Presenting this in prose requires massive cognitive load.
  • Asymmetry mismatch: Knowing targets is useless without context. The core question is: "Where does the asset trade relative to the risk envelope?" Current price versus target ranges must be instantly legible.

Solution Approach

The implementation integrates these artifacts cleanly across the existing Vue-TS frontend and the Pi agent backend:

  • Frontend (Vue-TS): Extend the App.vue component's ArtifactKind type union, write computed layout/normalization helpers for the three schemas, and construct clean, beautifully stylized templates for each inside the artifact pane. Define responsive styles in style.css.
  • Pi Agent Extension: Update steelman-product.ts to register the new literal values in ArtifactKind so the RPC schema validation accepts them.
  • System Prompting: Document the schemas and trigger criteria inside STEELMAN_AGENT_SYSTEM_PROMPT.md so the model learns the structural formats and emits them appropriately.
  • Validation Suite: Update the validation script to recognize and assert the new artifact kinds are healthy.

  2. Step 2: Update Script Assertions

    Define new interfaces, computed properties/methods for extracting quote data, timeline events, and gauge ranges with proportional position calculations.

  4. Step 4: App.vue Template Additions

  5. Step 5: Write Component Stylesheets

    Add precise CSS class rules to apps/steelman/web/src/style.css for proper padding, SVG arrow nodes, timeline chronological dashed pipelines, and high contrast highlighting of current valuations.

  6. Step 6: Revamp System Prompt Instructions

    Extend the system guidelines block in apps/steelman/extension/STEELMAN_AGENT_SYSTEM_PROMPT.md to define the target data shapes of the new artifact types with clear markdown guidelines.

  7. Step 7: Re-Build and Execute Tests

    Verify that running bun run dev compile finishes successfully and mock verification script passes completely.

Testing Strategy

Testing will be executed in three parts:

  • Static compilation checks: Vite TS compilation will verify typing and properties in the updated frontend build: 
    cd apps/steelman/web && bun run build
  • Schema validation script: Run the product-agent validation harness in mock mode: 
    cd apps/steelman && STEELMAN_MOCK=1 bun scripts/validate-steelman.ts
  • Visual layout review: Load the web frontend and trigger mock agents using the predefined starter triggers to view the rendering quality, responsiveness, and dark mode alignment.

Acceptance Criteria

Category Requirement Details Verification Method
Quote Artifact Displays verbatim quote in italic, large decorative quotation marks, speaker block with role, date, and clickable inline URL. Includes thesis commentary container. Render verification in web dashboard with active URL clicking test.
Timeline Artifact Renders dated milestones in horizontal chain from left to right. Nodes are color-coded based on severity: bull (green), neutral (blue/cyan), bear (red). Supports vertical layout on smaller screen wraps. Review responsive width layouts using Chrome DevTools.
Valuation Gauge Shows horizontal range containing Bull, Base, and Bear case price tags. Visually indicates where "Current trades today" sits on the relative range spectrum. Compare rendered position of indicator vs numerical coordinates to check math.
System Alignment The LLM Agent successfully parses the extended MD schema, selects the correct artifact types in tests, and emits standard compliant data structures. Run validation pipeline and check generated run snapshots under storage/runs/.
