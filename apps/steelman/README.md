# Steelman — Product Pi Agent Demo

A demo product application that embeds a Pi agent behind a polished investment-analysis UI while streaming the agent's internal lifecycle to the existing `pi-observability` server.

The product is an **Investment Steelman Agent**: give it an investment thesis and it argues the strongest counter-case. The right pane is a chat stream. The left pane renders structured artifacts generated by agent tools (`table`, `bar-chart`, `pie-chart`, `text`, `html`). Chat responses reference artifacts with `@refs`; clicking a ref brings that artifact into view.

## Architecture

```txt
apps/steelman/
├── server/                 Bun API + SSE + Pi RPC process manager
├── web/                    Vite Vue-TS product UI
├── extension/              Pi product extension: research + artifact tools
├── scripts/                validation helpers
├── storage/runs/           run snapshots (gitignored by convention if desired)
└── .sessions/              Pi session files for product-agent runs
```

Runtime flow:

1. Browser posts a thesis to `POST /api/runs`.
2. Bun backend creates a run and starts `pi --mode rpc` with two extensions:
   - top-level `extension/pi-observability.ts`
   - `apps/steelman/extension/steelman-product.ts`
3. The product extension adds:
   - `steelman_research` — tries Firecrawl CLI first, then falls back to dependency-light DuckDuckGo HTML web search
   - `steelman_emit_artifact` — posts structured UI artifacts back to this app
4. Pi observability streams all lifecycle/tool/message events to the observability server.
5. The Steelman app streams product chat/artifact events to the browser over `/api/runs/:id/stream`.
6. The UI links each run to the observability swimlane/single-session view via the configured `OBS_SERVER_URL` + token.

## Run locally

From the repo root, `just` boots everything with the right ports and a shared token:

```bash
just all              # obs server + Steelman backend + Steelman web
```

Open the Steelman UI at <http://127.0.0.1:51730>; watch the same run in the observability UI (the obs server prints a tokenized URL on boot). Run the pieces individually if you prefer:

```bash
just obs              # observability server       → http://127.0.0.1:43190
just steelman-server  # Steelman backend (Pi RPC)  → http://127.0.0.1:45210
just steelman-web     # Steelman Vite UI           → http://127.0.0.1:51730
```

## Validate

```bash
just validate-steelman
```

Spawns the backend and drives one real Pi-RPC run end-to-end — asserts chat `@refs`, ≥3 artifacts, and an accepted follow-up. Requires `pi` authenticated for the configured model.

## Environment

Backend:

| Var | Default | Purpose |
|---|---:|---|
| `STEELMAN_PORT` | `45210` | Bun API port |
| `STEELMAN_HOST` | `127.0.0.1` | Bun API host |
| `STEELMAN_PI_BIN` | `pi` | Pi executable |
| `STEELMAN_AGENT_MODEL` | `gemini-3.5-flash` | `--model` passed to Pi |
| `STEELMAN_AGENT_MODEL_PROVIDER` | `google` | `--provider` passed to Pi |
| `OBS_SERVER_URL` | `http://127.0.0.1:43190` | Observability server URL |
| `OBS_AUTH_TOKEN` | unset | Token forwarded to observability extension |
| `FIRECRAWL_TIMEOUT_MS` | `20000` | Research tool timeout |
| `FIRECRAWL_BIN` | `firecrawl` | Firecrawl executable; set to a missing name to force generic-search fallback in tests |
| `STEELMAN_AGENT_TIMEOUT_MS` | `180000` | Max real Pi agent run time before backend marks it errored |
| `STEELMAN_OBS_VIEW` | `single` | Observability deep-link view (`single` or `swimlane`) |

## Notes

This app intentionally has its own product SSE stream because product UX wants token deltas, artifact events, and run state in a product-native shape. The observability server remains the source of truth for internal Pi telemetry, debugging, and engineering review.
